## 检查清单
1. 查看最近 5 个 GitHub Actions 运行状态
2. 查看 http://phones.jiucai.eu.org/data/latest.json 的双源数据量
3. 检查爬虫工作流是否有报错
4. 检查合并分析工作流是否正常运行
5. 检查 Pages 是否部署成功

## ⚠️ 关键规则：任何代码修改必须走多模型评审
- 先用 `bash`/`gh`/`curl` 调查问题，不要凭猜测改代码
- 确定根因后，并行启动 3 个不同 Review AGENT 分析确认
- 综合评审结果确定修复方案后再实施
- 修复后必须再启动 3 个不同 Review AGENT 验证
- 全部通过后才提交推送
- **禁止单模型直接改代码后提交**

## 如果发现问题
按照 AGENTS.md 的项目规则：自动修复流程。调取工作流日志 → 并行多模型评审分析 → 修复 → 并行多模型评审验证 → 提交推送。
